DNS


DNS(Domain name server) is a global and highly distributed network service that resolves strings of letters into IP address. DNS uses UDP instead of TCP, but in complex situation DNS uses

DNS over TCP does in fact exist and is also in use all over. As the Web has gotten more complex, it's no longer the case that all DNS lookup responses can fit in a single UDP datagram. In these situations, a DNS name server would respond with a packet explaining that the response is too large. The DNS client would then establish a TCP connection in order to perform the lookup.

A domain name is the term we use for something that can be resolved by DNS.

Name Resolution: This process of using DNS to turn a domain name into an IP address is known as name resolution.

Anycast: Anycast is a technique that's used to route traffic to different destinations depending on factors like location, congestion, or link health

TTL in DNS: This is a value of time in seconds that can be configured by the owner of the domain name for how long a name server is allowed to cache an entry before it discard it and perform a full resolution again.


There are five types of DNS server:
• caching name servers,
• recursive name servers,
• root name servers, (13)
• TLD name servers (top lever domain) , and
• authoritative name servers

Now there are not actually 13 physical root name servers anymore rather 13 authorities that provide root name server lookup as a service.(using Anycast)

Caching and recursive name server are provided by ISP or local network provider.

For each TLD there is a TDL server. ( not necessary to be a physical name server)

Most important task is full recursive resolution:
1. Go to Root name server
2. root server refer to a certain TLD name server
3. TLD name server refer to authoritative name server
4. Authoritative name server provide the actual IP address of the requested domain name.
