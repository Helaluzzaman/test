TCP segment


It is like ethernet frame (in data link layer) or IP datagram ( In network layer)
Anatomy of TCP segment:

Control flag has 6 flag and are called TCP control flag (100110) : Urg(urgent), ack (acknowledgement), psh (Push), rst(Reset), syn (Synchronize), fin (Finish)
• URG
• ACK
• PSH
• RST
• SYN
• FIN

ACK detail : Acknowledgement number field should be examined.
SYN detail : It's used when first establishing a TCP connection and makes sure the receiving end knows to examine the sequence number field .
PSH (push) detail: This means, that the transmitting device wants the receiving device to push currently- buffered data to the application on the receiving end as soon as possible.This is for sending small amount of information which should not be buffered and needs imediate response and so getting imediate response from receiver.

TCP connection establishment: (three way hand-shake)
> syn (000010)
< syn/ack (010010)
> ack(010000)

TCP connection closing : (Four way Hand-shake)
> fin (000001)
< ack (010000)
< fin (000001)
> ack(010000)
