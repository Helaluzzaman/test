Resource record type


A record: An A record is used to point a certain domain name at a certain IPv4 IP address.

DNS round robin : If a domain name has multiple a record ( Ip address), ip addresses will be given to resolver computer in round robin pattern.
It is called DNS round robin logic. It is to be used to balance between multiple IPs those are resolved to a domain name.

Quad A record: It is like A record but it works with IPv6 address. It is becoming more and more popular.

CNAME: stands canonical name. It redirects traffic to one domain to another domain.

Explanation of cname : Let's say that Microsoft runs their web servers at www.microsoft.com. They also want to make sure that anyone that enters just microsoft.com into their web browser will get properly redirected. By configuring a CNAME record for microsoft.com that resolves to www.microsoft.com, the resolving client would then know to perform another resolution attempt, this time for www.microsoft.com, and then use the IP returned by that second attempt.

MX record stands for Mail Exchange record.
Explanation: Many companies run their web and mail servers on different machines with different IPs, so the MX record makes it easy to ensure that email gets delivered to a company's mail server, while other traffic like web traffic would get delivered to their web server.

SRV record - stands for service record. It is like MX record but it works for any service not only like MX record which only for mail service.

TXT record
Pointer resource record (PTR) : resolves an IP to a domain name. ( Reverse of A record)
